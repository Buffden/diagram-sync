# D2 Provider

Renders `.d2` files using the `d2` CLI. Output format is configurable — defaults to `svg`.

---

## Installation

### macOS

```bash
brew install d2
```

### Linux

```bash
curl -fsSL https://d2lang.com/install.sh | sh
```

### Windows

Download from [d2lang.com](https://d2lang.com/tour/install) and ensure `d2` is on your PATH.

---

## Supported Extensions

| Extension | Description |
| --- | --- |
| `.d2` | D2 diagram source file |

---

## Supported Output Formats

All formats are fully supported. Default is `svg`.

`svg` `png`

---

## Output

Images generated under `diagrams/`, mirroring the source path.

```text
src/architecture/system.d2  →  diagrams/src/architecture/system.svg
docs/flows/pipeline.d2      →  diagrams/docs/flows/pipeline.svg
```

---

## Config

No config required. D2 files are discovered and rendered automatically in `svg`.

To set a global output format:

```json
{
  "format": "png"
}
```

To set format per job:

```json
{
  "jobs": [
    {
      "name": "architecture",
      "type": "d2",
      "format": "png"
    }
  ]
}
```

To override at runtime:

```bash
npx diagram-sync --format png
```

Format resolution order: `--format` flag → job `format` → global `format` → default `svg`.

---

## CI/CD

Copy [`workflow.yml`](../../workflow.yml) into `.github/workflows/` in your repo. It generates images on every push and commits them back automatically.

Requires a Personal Access Token (PAT) with `contents: write` permission saved as a repository secret named `PAT_TOKEN`. [How to create a PAT](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens).

---

## Notes

- If `d2` is not found, `diagram-sync` skips D2 files with a warning and continues
- D2 files can be created by hand, generated by AI, or exported from a diagramming tool — `diagram-sync` renders whatever is committed
- For D2 language reference and layout options see the [D2 docs](https://d2lang.com/tour/intro)
