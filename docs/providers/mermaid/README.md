# Mermaid Provider

Renders `.mmd` and `.mermaid` files to SVG using the Mermaid CLI (`mmdc`).

---

## Installation

```bash
npm install -g @mermaid-js/mermaid-cli
```

Verify:

```bash
mmdc --version
```

---

## Supported Extensions

| Extension | Description |
| --- | --- |
| `.mmd` | Standard Mermaid source file |
| `.mermaid` | Alternate Mermaid extension |

---

## Output

SVG files generated under `diagrams/`, mirroring the source path.

```text
docs/flows/auth.mmd          →  diagrams/docs/flows/auth.svg
src/architecture/system.mmd  →  diagrams/src/architecture/system.svg
```

---

## Config

No config required. Mermaid files are discovered and rendered automatically.

To run Mermaid jobs explicitly via config:

```json
{
  "jobs": [
    {
      "name": "flows",
      "type": "mermaid"
    }
  ]
}
```

```bash
npx diagram-sync --config diagram-sync.config.json
```

## CI/CD Workflows

| Workflow | Description |
| --- | --- |
| [`workflow-generate.yml`](./workflow-generate.yml) | Generate images on push — files exist only in the runner |
| [`workflow-commit.yml`](./workflow-commit.yml) | Generate images and commit them directly to `main` |

Copy the relevant file into `.github/workflows/` in your repo.

## Notes

- If `mmdc` is not found, `diagram-sync` skips Mermaid files with a warning and continues
- Mermaid files can be created by hand, generated by AI, or exported from a diagramming tool — `diagram-sync` renders whatever is committed
- For Mermaid CLI configuration options (themes, backgrounds, etc.) see the [Mermaid CLI docs](https://github.com/mermaid-js/mermaid-cli)
