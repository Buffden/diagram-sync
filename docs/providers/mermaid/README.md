# Mermaid Provider

Renders `.mmd` and `.mermaid` files using the Mermaid CLI (`mmdc`). Output format is configurable — defaults to `png`.

---

## Installation

```bash
npm install -g @mermaid-js/mermaid-cli
```

Verify:

```bash
mmdc --version
```

---

## Supported Extensions

| Extension | Description |
| --- | --- |
| `.mmd` | Standard Mermaid source file |
| `.mermaid` | Alternate Mermaid extension |

---

## Supported Output Formats

| Format | Default |
| --- | --- |
| `png` | Yes |
| `svg` | — |
| `pdf` | — |

---

## Output

Images generated under `diagrams/`, mirroring the source path.

```text
docs/flows/auth.mmd          →  diagrams/docs/flows/auth.png
src/architecture/system.mmd  →  diagrams/src/architecture/system.png
```

---

## Config

No config required. Mermaid files are discovered and rendered automatically in `png`.

To set a global output format:

```json
{
  "format": "svg"
}
```

To set format per job:

```json
{
  "jobs": [
    {
      "name": "flows",
      "type": "mermaid",
      "format": "svg"
    }
  ]
}
```

To override at runtime:

```bash
npx diagram-sync --format pdf
```

Format resolution order: `--format` flag → job `format` → global `format` → default `png`.

---

## CI/CD

Copy [`workflow.yml`](../workflow.yml) into `.github/workflows/` in your repo. It generates images on every push and commits them back automatically.

Requires a Personal Access Token (PAT) with `contents: write` permission saved as a repository secret named `PAT_TOKEN`. [How to create a PAT](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens).

---

## Notes

- If `mmdc` is not found, `diagram-sync` skips Mermaid files with a warning and continues
- Mermaid files can be created by hand, generated by AI, or exported from a diagramming tool — `diagram-sync` renders whatever is committed
- For Mermaid CLI configuration options (themes, backgrounds, etc.) see the [Mermaid CLI docs](https://github.com/mermaid-js/mermaid-cli)
