# Draw.io Provider

Renders `.drawio` and `.dio` files using the Draw.io desktop CLI. Output format is configurable ‚Äî defaults to `svg`.

---

## Installation

### macOS

```bash
brew install --cask drawio
```

### Linux (Debian/Ubuntu)

```bash
DRAWIO_VERSION=$(curl -s https://api.github.com/repos/jgraph/drawio-desktop/releases/latest | jq -r '.tag_name | ltrimstr("v")')
wget -q "https://github.com/jgraph/drawio-desktop/releases/download/v${DRAWIO_VERSION}/drawio-amd64-${DRAWIO_VERSION}.deb" -O drawio.deb
sudo apt-get install -y ./drawio.deb
```

### Windows

Download from [draw.io](https://www.drawio.com) and ensure `drawio` is on your PATH.

## Supported Extensions

| Extension | Description |
| --- | --- |
| `.drawio` | Standard Draw.io source file |
| `.dio` | Alternate Draw.io extension |

---

## Supported Output Formats

All formats are fully supported. Default is `svg`.

`svg` `png` `jpg` `jpeg` `pdf`

---

## Output

Images generated under `diagrams/`, mirroring the source path.

```text
src/architecture/system.drawio  ‚Üí  diagrams/src/architecture/system.svg
docs/flows/pipeline.dio         ‚Üí  diagrams/docs/flows/pipeline.svg
```

---

## Config

No config required. Draw.io files are discovered and rendered automatically in `svg`.

To set a global output format:

```json
{
  "format": "png"
}
```

To set format per job:

```json
{
  "jobs": [
    {
      "name": "architecture",
      "type": "drawio",
      "format": "png"
    }
  ]
}
```

To override at runtime:

```bash
npx diagram-sync --format png
```

Format resolution order: `--format` flag ‚Üí job `format` ‚Üí global `format` ‚Üí default `svg`.

## CI/CD

Copy [`workflow.yml`](../workflow.yml) into `.github/workflows/` in your repo. It generates images on every push and commits them back automatically.

Requires a Personal Access Token (PAT) with `contents: write` permission saved as a repository secret named `PAT_TOKEN`. [How to create a PAT](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens).

## Notes

- If `drawio` is not found, `diagram-sync` skips Draw.io files with a warning and continues
- In CI environments (`CI=true`), `diagram-sync` automatically passes `--no-sandbox --disable-gpu` ‚Äî no manual workflow configuration needed. This is required because Draw.io is Electron-based and most CI runners do not support a display.
- Draw.io files can be created by hand, generated by AI, or exported from the Draw.io desktop or web app ‚Äî `diagram-sync` renders whatever is committed
- For Draw.io CLI options see the [Draw.io documentation](https://www.drawio.com/blog/diagrams-offline)
