# PlantUML Provider

Renders `.puml` and `.plantuml` files to SVG using the `plantuml` CLI.

---

## Installation

### macOS

```bash
brew install openjdk plantuml
```

### Linux (Debian/Ubuntu)

```bash
sudo apt-get update
sudo apt-get install -y --no-install-recommends default-jre plantuml
```

### Windows

Download from [plantuml.com](https://plantuml.com/download) and ensure `plantuml` is on your PATH.

---

## Supported Extensions

| Extension | Description |
| --- | --- |
| `.puml` | Standard PlantUML source file |
| `.plantuml` | Alternate PlantUML extension |

---

## Output

SVG files generated under `diagrams/`, mirroring the source path.

```text
src/services/auth/flow.puml   →  diagrams/src/services/auth/flow.svg
docs/architecture/system.puml →  diagrams/docs/architecture/system.svg
```

---

## Config

No config required. PlantUML files are discovered and rendered automatically.

To run PlantUML jobs explicitly via config:

```json
{
  "jobs": [
    {
      "name": "architecture",
      "type": "plantuml"
    }
  ]
}
```

```bash
npx diagram-sync --config diagram-sync.config.json
```

## CI/CD Workflows

| Workflow | Description |
| --- | --- |
| [`workflow-generate.yml`](./workflow-generate.yml) | Generate images on push — files exist only in the runner |
| [`workflow-commit.yml`](./workflow-commit.yml) | Generate images and commit them directly to `main` |

Copy the relevant file into `.github/workflows/` in your repo.

## Notes

- Requires Java 11+
- If `plantuml` is not found, `diagram-sync` skips PlantUML files with a warning and continues
- PlantUML files can be created by hand, generated by AI, or exported from a diagramming tool — `diagram-sync` renders whatever is committed
