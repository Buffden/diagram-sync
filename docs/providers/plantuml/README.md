# PlantUML Provider

Renders `.puml` and `.plantuml` files using the `plantuml` CLI. Output format is configurable — defaults to `svg`.

---

## Installation

### macOS

```bash
brew install openjdk plantuml
```

### Linux (Debian/Ubuntu)

```bash
sudo apt-get update
sudo apt-get install -y --no-install-recommends default-jre-headless plantuml
```

### Windows

Download from [plantuml.com](https://plantuml.com/download) and ensure `plantuml` is on your PATH.

---

## Supported Extensions

| Extension | Description |
| --- | --- |
| `.puml` | Standard PlantUML source file |
| `.plantuml` | Alternate PlantUML extension |

---

## Supported Output Formats

All formats are fully supported. Default is `svg`.

`svg` `png` `eps` `pdf`

---

## Output

Images generated under `diagrams/`, mirroring the source path.

```text
src/services/auth/flow.puml   →  diagrams/src/services/auth/flow.svg
docs/architecture/system.puml →  diagrams/docs/architecture/system.svg
```

---

## Config

No config required. PlantUML files are discovered and rendered automatically in `svg`.

To set a global output format:

```json
{
  "format": "svg"
}
```

To set format per job:

```json
{
  "jobs": [
    {
      "name": "architecture",
      "type": "plantuml",
      "format": "svg"
    }
  ]
}
```

To override at runtime:

```bash
npx diagram-sync --format pdf
```

Format resolution order: `--format` flag → job `format` → global `format` → default `svg`.

---

## CI/CD

Copy [`workflow.yml`](../../workflow.yml) into `.github/workflows/` in your repo. It generates images on every push and commits them back automatically.

Requires a Personal Access Token (PAT) with `contents: write` permission saved as a repository secret named `PAT_TOKEN`. [How to create a PAT](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens).

---

## Notes

- Requires Java 11+
- If `plantuml` is not found, `diagram-sync` skips PlantUML files with a warning and continues
- PlantUML files can be created by hand, generated by AI, or exported from a diagramming tool — `diagram-sync` renders whatever is committed
