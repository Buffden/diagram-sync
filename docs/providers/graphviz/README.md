# Graphviz Provider

Renders `.dot` and `.gv` files using the Graphviz `dot` CLI. Output format is configurable — defaults to `svg`.

---

## Installation

### macOS

```bash
brew install graphviz
```

### Linux (Debian/Ubuntu)

```bash
sudo apt-get update
sudo apt-get install -y graphviz
```

### Windows

Download from [graphviz.org](https://graphviz.org/download/) and ensure `dot` is on your PATH.

---

## Supported Extensions

| Extension | Description |
| --- | --- |
| `.dot` | Standard Graphviz source file |
| `.gv` | Alternate Graphviz extension |

---

## Supported Output Formats

All formats are fully supported. Default is `svg`.

`svg` `png` `eps` `pdf` `jpg` `jpeg` `gif`

---

## Output

Images generated under `diagrams/`, mirroring the source path.

```text
src/architecture/system.dot  →  diagrams/src/architecture/system.svg
docs/flows/pipeline.gv       →  diagrams/docs/flows/pipeline.svg
```

---

## Config

No config required. Graphviz files are discovered and rendered automatically in `svg`.

To set a global output format:

```json
{
  "format": "png"
}
```

To set format per job:

```json
{
  "jobs": [
    {
      "name": "architecture",
      "type": "graphviz",
      "format": "png"
    }
  ]
}
```

To override at runtime:

```bash
npx diagram-sync --format png
```

Format resolution order: `--format` flag → job `format` → global `format` → default `svg`.

---

## CI/CD

Copy [`workflow.yml`](../workflow.yml) into `.github/workflows/` in your repo. It generates images on every push and commits them back automatically.

Requires a Personal Access Token (PAT) with `contents: write` permission saved as a repository secret named `PAT_TOKEN`. [How to create a PAT](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens).

---

## Notes

- If `dot` is not found, `diagram-sync` skips Graphviz files with a warning and continues
- Graphviz files can be created by hand, generated by AI, or exported from a diagramming tool — `diagram-sync` renders whatever is committed
- For Graphviz layout engines and options see the [Graphviz docs](https://graphviz.org/documentation/)
